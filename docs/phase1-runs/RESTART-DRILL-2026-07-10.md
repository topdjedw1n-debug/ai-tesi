## Три знахідки ДО рестарту (сам дриль окупився)

Сім спроб (документи 52–58) впали на чесних гейтах — і кожне падіння
виявило реальну ваду, яка інакше з'їла б перший пілот:

1. **Гейт джерел перевіряв уже сконвертований текст** (регресія). Генератор
   замінює внутрішні маркери [Key] на готові APA-цитати, а гейт шукав саме
   маркери → нуль збігів → «no grounded citation» на кожному розділі.
   Фікс: письменницька чернетка з маркерами зберігається окремим полем
   (`content_with_markers`), гейт оцінює її. Регресійні тести додано.
2. **Граматичний гейт рахував markdown за помилки.** Заголовки `#`/`##`,
   списки і жирний текст LanguageTool читає як прозу → 29–38 «помилок» на
   чистій італійській. Фікс: розмітка зрізається перед перевіркою (текст
   лишається).
3. **Орфографічний словник LanguageTool топив сигнал.** MORFOLOGIK_RULE_IT_IT
   давав 33 з 35 «помилок» — усі на англійські технічні терміни
   («machine learning», «cloud computing»). Фікс: категорія TYPOS вимкнена
   конфігурованим параметром (LANGUAGETOOL_DISABLED_CATEGORIES, дефолт
   "TYPOS"); справжня граматика/пунктуація лишаються гейтом. Причина відмови
   тепер називає топ-правила — наступна така загадка читається з одного лога.

Попутно: після DDL-міграцій пул asyncpg тримає застарілі prepared
statements — воркер сипав помилками до рестарту процесу. На проді після
міграцій **обов'язковий рестарт API** (і так планувався).

## Хронологія kill → recovery (документ 59, job 48)

| Час | Подія |
|---|---|
| T0 | Генерація йде: розділи 1 і 2 закомічені, пишеться 3-й |
| T0 (18:32:03) | `kill -9` процесу API. Ліза мертвого власника (pid 9756) жива до 15:33:52 UTC |
| T0+~5s | Новий процес API піднято (pid 11834). Чужу живу лізу він **не краде** — чекає |
| T0+~70s | Ліза спливла → `claimed generation job 48 (attempt 2/3)` |
| T0+~75s | `♻️ Resuming generation from section 3/3` — розділи 1–2 НЕ перегенеровуються |
| T0+~110s | Розділ 3 закомічений |
| T0+~150s | Панель, верифікація цитат (strict), аудит тверджень (30 claims: 26 supported, 4 uncertain), експорт → **job completed, документ completed** |

Перевірки після відновлення:

- Дублі розділів: `GROUP BY section_index HAVING count(*)>1` → **порожньо**.
- Розділи: 3/3 completed (483/502/439 слів), заголовки італійські, за темою.
- Артефакт: один DOCX у сховищі, `sha256=fd24a5da7f8841d9…` прив'язаний до
  документа.
- Історія: одна робота, attempts=2, success=true.
- Провенанс-хвіст: panel_review → verification_summary → citation_gate →
  claims_verified ×3 → claim_check_summary → exported.
