# Експеримент MODEL-MATRIX: писар × гуманізатор (03.07.2026)

**Статус:** Фаза 1 IN PROGRESS
**Авторизація:** фаундер 03.07.2026 — «один пише, другий гуманізує,
поміксувати 4-5 моделей Anthropic × моделі GPT, прогнати, зробити папку
робіт, читаю вручну + віддаю GPT-5.5 на висновки; вибираємо найкращий
варіант по грошах і по humanization».

## Чому цей експеримент

Док 10 (Opus-писар) чесно зафейлив AI-детект: 42–70% проти порога 35,
причому гуманізація тією ж моделлю ПОГІРШУВАЛА скор (53.6→70.3).
Дві механічні знахідки, які робили попередні спроби гуманізації фіктивними:

1. **Гуманізатор завжди успадковує модель писаря** — та сама модель
   переписує свій текст у свій же стиль → детектор бачить ще типовіший
   почерк (generator.py / background_jobs.py передають provider/model
   писаря в Humanizer).
2. **«Прогресивна температура» multi-pass мертва на Claude 4+/5** —
   ці моделі не приймають temperature (humanizer шле її лише для
   claude-3*), тож 2-3 «різні» паси на Opus/Sonnet — це буквально
   той самий виклик повторно.

Гіпотеза: крос-модельна гуманізація (інша родина моделей) ламає
статистичний відбиток писаря; вибір комбо калібрувати ПО ДЕТЕКТОРУ
із запасом (ціль ≤25, не «впритул до 35»).

## Фаза 4 — детектор-у-циклі: best-of-N + заморозка цитат (03.07.2026)

**Що тестуємо.** Не «інша модель», а інший МЕТОД (наслідок Фази 3):

рятувальник gpt-4 для обох; ціль 30, поріг ≤35, ідеал ≤25.
- **A_serial:** `HUMANIZER_BEST_OF_N=1`, freeze OFF (legacy).

3. **Заморозка цитат = безпечна, але «лоссі».** gpt-4 місцями губить

4. **Якість не впала** ніде (панель 90.7–94.6) — трикутник не
   схлопнувся на цих входах.

**Чесні межі.**
- B має більше «пострілів» (9 vs 3), тож виграш Opus поєднує ширину
  best-of-N і freeze — не ізольовано. Чистий контроль: B' = best-of-N з
  freeze OFF (ще не робили).
- Свіжі входи вийшли важчі за історичні (~95 vs ~70) через терсовий
  source-pack; A/B валідний (той самий вхід), але абсолюти не рівняти
  до doc 19/20 напряму.
- n=1 секція на писаря; sonnet-5 може бути й дисперсією (але збігається
  з doc 20).

**Наступні кроки (потребують окремого «так» — платні):**
- ізолювати freeze vs best-of-N (B' freeze OFF);
- зробити сентінел стійкішим до gpt-4 (менше згорілих варіантів → ширша
